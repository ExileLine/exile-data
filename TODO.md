# AI 问数系统建设 TODO

## 设计判断

当前阶段可以不建设传统 BI 的图表、仪表盘和报表编排层，直接面向“自然语言问题 -> 语义匹配 -> SQL 计划 -> 安全校验 -> 查询执行 -> 结果解释”的 AI 问数闭环。

不能省略的底座：

- 数据源管理：保存只读连接信息、数据库类型、扩展配置。
- 语义层：告诉 AI 哪些表、字段、指标、维度、业务术语可以使用。
- SQL 安全：生成 SQL 前后都必须做只读、多语句、DDL/DML 风险校验。
- 问答审计：记录问题、候选语义模型、生成 SQL、风险等级、执行状态和错误。
